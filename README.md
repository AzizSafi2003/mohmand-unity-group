# Mohmand Unity Group — Community Platform

A bilingual (English / پښتو) community platform for a family network: family
directories, multi‑generation **family trees**, a registration **approval
workflow**, monthly **contributions** tracked on the **Afghan Solar Calendar**,
a transparent **financial system** with **PDF + email reports**, announcements,
community laws, a CMS‑driven homepage, a global search, and a full admin
dashboard — all behind role‑based access control (visitor / member / admin)
with right‑to‑left support throughout.

Built with **Next.js (App Router) + React 19 + TypeScript**, **Convex** (database

- serverless functions), and **Clerk** (authentication), styled with Tailwind in
  a custom Pashtun‑heritage design system.

---

## ⚠️ Read this first — what you are getting

This repository is a **complete, well‑architected foundation**: every screen,
every Convex function, the data model, the design system, i18n, and the build
tooling are real and written to production conventions. It is **not** a
turn‑key binary you can run untouched, because two of its pillars — Convex and
Clerk — are external services that must be provisioned with **your own
accounts and keys**. Until you do the setup below, the app will not start and
your editor will show type errors against `@convex/_generated/*` (that folder
is generated by Convex on first run — its absence is expected, not a bug).

A frank, itemised assessment of what is finished, what is partial, and what
still needs manual verification lives in
[`docs/PROJECT_AUDIT_REPORT.md`](docs/PROJECT_AUDIT_REPORT.md). Please read it.

## Quick start

> Prerequisites: **Node.js ≥ 18.18**, npm, and free **Convex** + **Clerk**
> accounts.

```bash
# 1. Install dependencies
npm install

# 2. Provision Convex (creates your deployment AND generates convex/_generated)
npx convex dev          # leave running in its own terminal

# 3. Configure environment
cp .env.example .env.local
#   then fill in the Convex URL (printed by step 2) and your Clerk keys

# 4. Run the web app (in a second terminal)
npm run dev

# 5. Build for production
npm run build
```

The build process now regenerates Convex bindings automatically before compiling.

Open <http://localhost:3000>. The first time you sign up with the email set in
`BOOTSTRAP_ADMIN_EMAIL`, that account is automatically promoted to an approved
admin (everyone else starts as a pending member awaiting approval).

The full, step‑by‑step version — including the **Clerk JWT template** and the
**Clerk → Convex webhook** — is in
[`docs/INSTALLATION_GUIDE.md`](docs/INSTALLATION_GUIDE.md). Do not skip it; the
JWT template and webhook are what make auth and approvals actually work.

├─ src/
│ ├─ app/ # Routes (public, member, /admin/_, /api/\*)
│ ├─ components/ # ui/ primitives · layout/ · shared/
